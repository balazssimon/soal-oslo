What is SOAL?

SOAL is a user friendly domain specific language with a familiar C#/Java-like syntax. SOAL is created to help automatic generation of XSD and WSDL files, and also projects with configuration files for the most popular SOA development environments. SOAL also supports WS-* standards.

SOAL is not a runtime environment for web services. SOAL is an offline tool for generating the necessary XSD, WSDL and configuration files to make top-down development of web services easier. Crafting WSDL interfaces by hand and matching the configuration options of the various SOA development environments for WS-* standards is very hard. Hence, the SOAL framework can greatly increase the development productivity of web services.

The SOAL framework supports the following standards:

  • SOAP (1.1 and 1.2)
  • WSDL (1.1)
  • WS-Addressing
  • WS-ReliableMessaging
  • WS-Security
  • WS-SecureConversation
  • WS-AtomicTransaction

The SOAL framework supports code generation for the following SOA development tools:

  • Microsoft Visual Studio, IIS server, Windows Communication Foundation (WCF)
  • Eclipse, JBoss Application Server, WildFly, Apache CXF, JAX-WS
  • Eclipse, Tomcat Server, Apache CXF, JAX-WS
  • IBM RAD, WebSphere Application Server, JAX-WS
  • Oracle JDeveloper, WebLogic Server, JAX-WS

The current SOAL implementation is based on the deprecated Microsoft Oslo metamodeling framework, but it will be ported to the lightweight MetaDslx framework.

Related projects

MetaDslx
MetaDslx is a lightweight metamodeling framework for .NET and Java. It helps implementing domain specific languages.

MetaDslx makes it easier to create compilers for the textual concrete syntax using ANTLR4. MetaDslx extends the ANTLR4 grammars with annotations on rules and elements, which make semantic parsing easier. MetaDslx also comes with a user friendly template-based code generator.
